## Анализ результатов

### Малые объёмы данных (2 000 – 8 000)

На небольших размерах данных CPU работает быстрее или сопоставимо с GPU.

Например:
- 2 000 элементов: GPU значительно медленнее (0.0023 сек против 0.00017 сек)
- 4 000 элементов: GPU уже начинает догонять CPU
- 8 000 элементов: GPU становится быстрее

**Причина:** накладные расходы GPU:
- копирование данных CPU → GPU
- запуск CUDA-ядра

---

### Средние объёмы данных (16 000 – 128 000)

GPU начинает стабильно выигрывать у CPU.

- 16 000: GPU ≈ в 6 раз быстрее  
- 128 000: GPU ≈ в 30 раз быстрее  

Параллелизм начинает эффективно компенсировать накладные расходы.

---

### Большие объёмы данных (256 000 и выше)

GPU показывает значительное преимущество.

Примеры:
- 1M элементов:  
  - CPU: ~0.10 сек  
  - GPU: ~0.0014 сек  
  - ускорение ≈ **70x**

- 8M элементов:  
  - CPU: ~0.69 сек  
  - GPU: ~0.0085 сек  
  - ускорение ≈ **80x**

С ростом объёма данных разница становится кратной.

---

## Итог

Графический процессор (GPU) значительно превосходит CPU при работе с большими массивами данных благодаря массовому параллелизму.

Однако при малых объёмах данных использование GPU неэффективно из-за накладных расходов на:

- копирование данных между CPU и GPU  
- запуск CUDA-ядра  

---

## Вывод

Оптимальная область применения GPU:

- большие массивы данных  
- задачи с высокой степенью параллелизма  
- вычисления, где можно эффективно распараллелить операции  

CPU остаётся более эффективным для небольших задач с низкими накладными расходами.
