# Перемножение матриц: CPU vs GPU (PyCUDA)

В работе сравнивается производительность умножения квадратных матриц на CPU и GPU (CUDA через PyCUDA).

## Результаты измерений

| Размерность | GPU (сек) | CPU (сек) | Ускорение |
| :---------: | :-------: | :-------: | :-------: |
|   50×50     | 0.0001    | 0.0499    | ~573x     |
|  100×100    | 0.0001    | 0.3792    | ~4762x    |
|  200×200    | 0.0001    | 2.8524    | ~20952x   |
|  400×400    | 0.0005    | 25.1882   | ~46850x   |
|  800×800    | 0.0045    | 203.3268  | ~45651x   |
| 1600×1600   | 0.0356    | 1638.6812 | ~46073x   |
| 3200×3200   | —         | —         | —         |
| 6400×6400   | —         | —         | —         |
|12800×12800  | —         | —         | —         |

> Для размеров 3200×3200 и выше вычисления не были завершены из-за ограничений времени выполнения и ресурсов среды.

---

## Характеристики системы

Тестирование выполнялось в Google Colab:

- CPU: виртуализированное окружение (обычно Intel Xeon)
- GPU: NVIDIA T4  
- Технологии: CUDA, PyCUDA, NumPy  

---

## Анализ результатов

### Малые размеры (50×50 — 100×100)

- ускорение от ~500x до ~4700x  
- время GPU практически не меняется (~0.0001 сек)

Причина:
- CPU реализован через тройной цикл (медленно)
- нагрузка на GPU минимальна

**Вывод:** для малых размеров разница есть, но сравнение не полностью корректно.

---

### Средние размеры (200×200 — 800×800)

- CPU: рост времени с ~2.8 сек до ~203 сек  
- GPU: рост с 0.0001 до 0.0045 сек  
- ускорение до ~45 000x  

**Вывод:** GPU начинает эффективно масштабироваться.

---

### Большие размеры (1600×1600)

- CPU: ~1638 секунд (~27 минут)  
- GPU: ~0.036 сек  

**Вывод:** CPU становится практически неприменим.

---

### Ограничения (3200×3200 и выше)

Причины:
- слишком долгое выполнение CPU
- ограничения среды (Colab timeout)
- возможные ограничения памяти GPU

---

## Ключевые наблюдения

- CPU:
  - сложность O(n³)
  - не оптимизирован (чистый Python)

- GPU:
  - высокая степень параллелизма
  - хорошая масштабируемость

- ускорение:
  - достигает десятков тысяч раз
  - частично завышено из-за слабой CPU-реализации

---

## Итоговые выводы

- GPU значительно эффективнее CPU при росте размерности  
- CPU подходит только для малых задач  
- GPU — основной инструмент для вычислений с большими матрицами  

---

## Заключение

Эксперимент демонстрирует преимущество параллельных вычислений на GPU над последовательными вычислениями на CPU при решении задач линейной алгебры.
